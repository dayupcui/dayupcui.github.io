前言

最近由于项目原因需要加入本地推送，我就把本地推送相关的API官方文档看了一遍，说实话，苹果的官方文档有些描述确实不好理解，关键是描述的太模糊，不够详细，接下来我就把本地推送相关的知识梳理一下。

本地推送(UILocalNotification)

本地推送是相对于远程推送来说的，苹果为什么要加入本地推送，这里要说一下它的特点：

  • 本地推送不需要服务器、由系统根据时间或者地点发出推送消息，展示形式和远程推送是一样的。
  • 本地推送数量是有限制的，最多只能安排和展示64个，超出的通知都会被废弃
  • 本地推送由开发者设定时间、进入特定的地点或者iBeacon区域时触发，基于地点的推送是iOS8之后加入的，iOS8系统还加入可以指定用户触发的操作（Actions），甚至不需要启动app就可以处理。

基于时间的本地推送

基于时间的本地推送是指：本地推送消息的触发是提前设定好时间，系统根据触发时间来推送消息。创建一个本地推送如下：

UILocalNotification *locNotification = [[UILocalNotification alloc] init];
locNotification.fireDate = [[NSDate date] dateByAddingTimeInterval:5];
locNotification.timeZone = [NSTimeZone defaultTimeZone];
locNotification.repeatInterval = 0;
locNotification.repeatCalendar = [NSCalendar currentCalendar];
locNotification.hasAction = YES;
locNotification.alertAction = @"打开";
locNotification.alertTitle = @"按时间推送";
locNotification.soundName =    UILocalNotificationDefaultSoundName;
locNotification.applicationIconBadgeNumber = 1;
locNotification.userInfo = @{@"key":@"name"};
locNotification.category = @"alert";
[[UIApplication sharedApplication] scheduleLocalNotification:locNotification];

属性说明：

  • fireDate — 触发推送的日期，如果设定为nil或者过去的时间，则立即触发。
  • timeZone — 时区，fireDate会根据不同的时区调整。
  • repeatInterval — 重复触发的时间间隔(NSCalendarUnit类型),意味着你不能随意设定，不能设置小于1分钟的间隔，否则会发生异常
  • repeatCalendar — 重复触发的时间间隔依据的日历设定
  • hasAction — 是否显示动作按钮
  • alertAction — 动作按钮的标题，在屏幕锁屏的时候，跟在“滑动来”文字之后
  • alertTitle — 通知的标题，一般在通知中心中显示通知的时候出现
  • soundName — 设置推送的声音，不设置就没有声音，系统默认声音为UILocalNotificationDefaultSoundName，自定义声音格式为wav,caf或者aiff，苹果官方没有写MP3，但是我设置MP3格式依然能用。声音长度不能超过30s，如果超过30s，则被系统默认声音取代。
  • applicationIconBadgeNumber — 应用icon右上角的badge。
  • userInfo — 通过推送可以传递一些信息
  • category — 一般在iOS8系统之上使用，表示一些action组，是UIUserNotificationCategory的identifier值。

主要讲一下repeatInterval 这个属性

repeatInterval 不可以随意设置任何值，它是NSCalendarUnit枚举类型，所以只能设置一些固定的值，可以看一下它所包含的枚举类型如下：

  • NSCalendarUnitEra 表示一个纪元，跟不同的日历相关，比如格林标准日历使用了公元前和公元后作为它的两个纪元。
  • NSCalendarUnitYear 每年
  • NSCalendarUnitMonth 每月
  • NSCalendarUnitDay 每天
  • NSCalendarUnitHour 每小时
  • NSCalendarUnitMinute 每分钟
  • NSCalendarUnitSecond 每秒
  • NSCalendarUnitWeekday 星期几 1-7
  • NSCalendarUnitWeekdayOrdinal 这个月第几个星期几 比如：2月份第3个星期三
  • NSCalendarUnitQuarter 一刻钟（15分钟）
  • NSCalendarUnitWeekOfMonth 一个月的第1周
  • NSCalendarUnitWeekOfYear 一年内的第1周
  • NSCalendarUnitYearForWeekOfYear 这年的这一周所关联的年份
  • NSCalendarUnitNanosecond Nano的每秒
  • NSCalendarUnitCalendar 日历的日历，具体没闹明白
  • NSCalendarUnitTimeZone 日历所在的时区

假如你真想设置一个类似于5分钟这样的间隔，该这么做呢？办法只能是创建3个fireDate相差5分钟、repeatInterval为NSCalendarUnitQuarter的本地通知，这样就可以建立起循环。其他想自定义的间隔都可以按照这个思路来，只要限制在64个通知范围内就可以。

基于地点的本地推送

基于地点的推送是在iOS8系统引入的一项功能，我们需要在app中设置好一个坐标（经纬度）和一个半径（范围），当装有app的设备进入这个范围时，系统就会发出一个消息通知。创建一个本地推送步骤如下：

  1. 登记位置信息，获取用户的授权

    CLLocationManager *locMan = [[CLLocationManager alloc] init];
locMan.delegate = self;
[locMan requestWhenInUseAuthorization];

    在info.plist中加入如下图:

    当程序运行到“[locMan requestWhenInUseAuthorization];”这一句时会弹出对话框:

  2. 获取用户权限的回调

    - (void)locationManager:(CLLocationManager *)manager
didChangeAuthorizationStatus:(CLAuthorizationStatus)status 
{
       // check status to see if we’re authorized
       BOOL canUseLocationNotifications =
     (status == kCLAuthorizationStatusAuthorizedWhenInUse);
       if (canUseLocationNotifications) {
           [self startShowingLocationNotifications];
    }
}

  3. 注册本地推送

    #define LOC_COORDINATE CLLocationCoordinate2DMake(40.2781921, 100.752936)
#define LOC_RADIUS 100
#define LOC_IDENTIFIER @"identifier"

- (void)startShowingLocationNotifications 
{
    UILocalNotification *locNotification = [[UILocalNotification alloc]
                                   init];
    locNotification.alertBody = @“You have arrived!”;
    locNotification.regionTriggersOnce = YES;
    locNotification.region = [[CLCircularRegion alloc]
                   initWithCenter:LOC_COORDINATE
                           radius:LOC_RADIUS
                       identifier:LOC_IDENTIFIER];
    [[UIApplication sharedApplication]
                 scheduleLocalNotification:localNotification];
}

  4. 监控地理位置

    如果你想监控设备是否进入或者离开设定的区域，有相关的代理方法，

    //进入该区域
-(void)locationManager:(CLLocationManager *)manager didEnterRegion:(CLRegion *)region
{
        NSLog(@"%s",__func__);
}

//离开该区域
- (void)locationManager:(CLLocationManager *)manager didExitRegion:(CLRegion *)region
{
    NSLog(@"%s",__func__);
}

iOS8推送新特性

在ios8之后，注册推送之前必须调用[UIApplicationregisterUserNotificationSettings:]方法。其中引入了一些新特性，说明如下：

  • 用户收到推送通知可以自定义一些动作UIUserNotificationAction
  • 每个推送都可以设置一组动作，使用UIUserNotificationCategory
  • 这些动作分组在应用启动时就需要注册到UIUserNotificationSettings

用户动作(UIUserNotificationAction)

用户行为是app收到推送后，可以操作的一些按钮和输入框，这里讲一下UIUserNotificationAction都包含哪些属性：

  • identifier 动作的唯一标识符，可以通过它获取动作本身。
  • title 动作按钮的title
  • behavior(iOS9引入的) 是个UIUserNotificationActionBehavior的枚举类型。包含两个枚举值：
    UIUserNotificationActionBehaviorDefault 是默认行为。
    UIUserNotificationActionBehaviorTextInput 显示一个输入框，用户可以输入内容。
  • parameters(iOS9引入的) 可以设定Action的一些属性类型。举例：可以改变输入框右边button的标题，只要把[NSDictionary dictionaryWithObject:@”评论” forKey:UIUserNotificationTextInputActionButtonTitleKey]赋值给parameters就可以。
  • activationMode 点击或者滑动推送打开应用的模式，是个UIUserNotificationActivationMode枚举类型，包含UIUserNotificationActivationModeForeground：打开展示应用。
    UIUserNotificationActivationModeBackground：不打开应用或者在后台显示。这两种枚举值
  • authenticationRequired 进入应用之前是否解锁设备，如果activationMode为UIUserNotificationActivationModeForeground模式，则这个属性可以忽略。否则需要判断是否解锁。
  • destructive 动作按钮显示为红色背景，起到警示的作用。

注意创建动作时要使用UIMutableUserNotificationAction这个类

动作分组(UIUserNotificationCategory)

可以把创建的一系列动作加入一个分组中，创建分组需要调用UIMutableUserNotificationCategory中的方法：

- (void)setActions:(nullable NSArray<UIUserNotificationAction *> *)actions forContext:(UIUserNotificationActionContext)context

其中 actions为一系列动作的数组。
UIUserNotificationActionContext包含两个枚举值：UIUserNotificationActionContextDefault 表示推送通知最多能显示4个按钮
UIUserNotificationActionContextMinimal 表示推送通知最多能显示2个按钮

属性
identifier：分组的唯一标识符，一般用在UILocalNotification的Category。

通知设置(UIUserNotificationSettings)

把创建的分组加入到UIUserNotificationSettings中，需要调用方法： 

+ (instancetype)settingsForTypes:(UIUserNotificationType)types
                  categories:(nullable NSSet<UIUserNotificationCategory *> *)categories;

其中types为UIUserNotificationType枚举类型。categories为一系列分组。

两个Key值

  • UIUserNotificationTextInputActionButtonTitleKey 设置输入框右边action按钮的标题。
  • UIUserNotificationActionResponseTypedTextKey action触发回调，可以通过该值获取输入框的内容。

举例说明

下面举例来说明新特性如何使用。

不带输入框

- (void)registerLocalNotificationSettings
{
       UIMutableUserNotificationAction *action1 = [[UIMutableUserNotificationAction alloc] init];
    action1.identifier = @"action1";
    action1.title = @"接受";
    action1.behavior =     UIUserNotificationActionBehaviorDefault;
    action1.activationMode = UIUserNotificationActivationModeForeground;

    UIMutableUserNotificationAction *action2 = [[UIMutableUserNotificationAction alloc] init];
    action2.identifier = @"action2";
    action2.title = @"拒绝";
    action2.behavior = UIUserNotificationActionBehaviorDefault;
    action2.activationMode = UIUserNotificationActivationModeBackground;
    action1.authenticationRequired = YES;
    action1.destructive = YES;

    UIMutableUserNotificationCategory *category = [[UIMutableUserNotificationCategory alloc] init];
    category.identifier = @"alert";
    [category setActions:@[action1,action2] 
    forContext:UIUserNotificationActionContextMinimal];

    UIUserNotificationType type = UIUserNotificationTypeBadge | UIUserNotificationTypeSound | UIUserNotificationTypeAlert;
    UIUserNotificationSettings *settings = [UIUserNotificationSettings settingsForTypes:type categories:[NSSet setWithObjects:category, nil]];
    [[UIApplication sharedApplication] registerUserNotificationSettings:settings];
}

创建一个基于时间的本地推送,然后把UILocalNotification的category赋值为”alert”，效果图如下:

点击“接受”和“拒绝”，会调用代理回调方法如下：

- (void)application:(UIApplication *)application handleActionWithIdentifier:(nullable NSString *)identifier forLocalNotification:(UILocalNotification *)notification completionHandler:(void(^)())completionHandler
{
    if ([identifier isEqualToString:@"action1"]) {
        NSLog(@"接受");
    }
    else if ([identifier isEqualToString:@"action2"])
    {
        NSLog(@"拒绝");
    }

    if(completionHandler)
    {
        completionHandler();
    }
}

根据action不同的identifier来做出相应的处理，这里“action1”表示“接受”按钮被点击，“action2”表示“拒绝”按钮被点击。

这里要说明一下，你会发现不管是锁屏状态或者是顶部Banner条，都只能最多显示两个按钮，那么四个按钮该如何显示呢？需要你把推送显示效果变成弹出框的形式，如下图：

操作步骤如下：打开设置–>通知–>应用本身->把“横幅”显示改成“提醒”显示，这样就OK了。

带输入框

- (void)registerLocalNotificationSettings
{
       UIMutableUserNotificationAction *action1 = [[UIMutableUserNotificationAction alloc] init];
    action1.identifier = @"action1";
    action1.parameters = @{UIUserNotificationTextInputActionButtonTitleKey:@"评论"};
    action1.title = @"评论";
    action1.behavior =     UIUserNotificationActionBehaviorDefault;
    action1.activationMode = UIUserNotificationActivationModeForeground;

    UIMutableUserNotificationAction *action2 = [[UIMutableUserNotificationAction alloc] init];
    action2.identifier = @"action2";
    action2.title = @"拒绝";
    action2.behavior = UIUserNotificationActionBehaviorDefault;
    action2.activationMode = UIUserNotificationActivationModeBackground;
    action1.authenticationRequired = YES;
    action1.destructive = YES;

    UIMutableUserNotificationCategory *category = [[UIMutableUserNotificationCategory alloc] init];
    category.identifier = @"alert";
    [category setActions:@[action1,action2] 
    forContext:UIUserNotificationActionContextMinimal];

    UIUserNotificationType type = UIUserNotificationTypeBadge | UIUserNotificationTypeSound | UIUserNotificationTypeAlert;
    UIUserNotificationSettings *settings = [UIUserNotificationSettings settingsForTypes:type categories:[NSSet setWithObjects:category, nil]];
    [[UIApplication sharedApplication] registerUserNotificationSettings:settings];
}

需要创建一个本地通知UILocalNotification，然后把它的category属性赋值为“alert”，效果图如下

点击“评论“按钮，效果变成如下：

这个时候会出现输入框，让用户输入内容，输入完内容，点击右边的”评论“，应用调用回调方法如下:

- (void)application:(UIApplication *)application handleActionWithIdentifier:(NSString *)identifier forLocalNotification:(UILocalNotification *)notification withResponseInfo:(NSDictionary *)responseInfo completionHandler:(void (^)())completionHandler
{
    if ([identifier isEqualToString:@"action1"]) {
        NSLog(@"评论");
        NSString *msg = responseInfo[UIUserNotificationActionResponseTypedTextKey];
        NSLog(@"msg = %@",msg);
    }
    else if ([identifier isEqualToString:@"action2"])
    {
            NSLog(@"拒绝");
    }

    if(completionHandler)
    {
        completionHandler();
    }
}

在代理回调方法中可通过UIUserNotificationActionResponseTypedTextKey这个键获取用户输入的内容。
