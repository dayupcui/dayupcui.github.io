前言

以前写博客都是在CSDN这些技术网站上面，感觉样式很固定，后来看了一些大神的技术博客，他们的博客界面简洁，定制化程度高。原来他们都是用github搭配hexo来做的专属于自己的博客，顿时让我觉得热血沸腾，写博客还可以这么搞，自己也就动手搭建了一个。

安装

因为Hexo是静态博客，所以首先得在本地上搭建，搭建之前需要安装Node.js和Git。

  • 安装Node.js官网下载最新版本，一路安装即可，安装完成后就可以使用node和npm命令。
  • 安装Git，MAC系统Xcode安装后会自带Git，其他系统可以参考网上教程安装。
  • 在Github上创建账号，用Github来做博客的远程仓库，与本地的Hexo连接。

下面开始正式安装Hexo，打开终端输入：

npm install -g hexo

创建博客目录

我们创建一个文件夹来作为博客目录，比如：MyBlog，然后执行命令

$ cd MyBlog
$ hexo init
$ npm install

命令执行完后，会在MyBlog文件夹中生成如下目录结构：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   └── _posts
└── themes

Hexo的目录结构简单：

  • _config.yml 是博客的配置文件
  • scaffolds 是博客文章模板
  • source 是博客文章目录
  • themes 存放主题风格文件

启动本地服务

安装完成后，我们可以预览一下效果，在终端中执行命令（所有命令必须在博客目录下）

hexo server(或者hexo s)

等服务启动之后，打开浏览器，输入http://localhost:4000,这个时候你就会看到一个简陋的页面，这就是hexo的初始页面，也是你的博客小窝。简陋不要紧，hexo有很多主题供你选择。

建立Github关联

如果想要通过github地址来访问你的博客，你需要建立github和hexo的关联，具体步骤如下：

  1. 在github上建立与你用户名对应的仓库【username.github.io】，比如远程仓库地址为：https://github.com/dayupcui/dayupcui.github.io.git，或者 git@github.com:dayupcui/dayupcui.github.io.git（需要创建ssh key）

  2. 在博客目录中找到_config.yml文件，在终端用vim _config.yml命令打开，找到最下面，改为

    deploy: 
type: git
repository: git@github.com:dayupcui/dayupcui.github.io.git
branch: master

  3. 在终端执行命令：

    hexo deploy
  4. 在浏览器中输入http://dayupcui.github.io/就可以访问到你的博客了，是不是很兴奋啊！！！ 但一定要记住账户名改成你的。

更换主题

Hexo的默认主题是不是很丑，是不是很想换一个漂亮的主题、或者是很有个性的主题。只要在github中输入“hexo theme”就可以搜索到很多主题。我自己是比较喜欢简洁风格的主题，所以就选了jacman，这里需要提一下唐巧大神，我也是参照他的博客风格，把jacman主题重新定制了一下。下面就以jacman主题为列子说一下如何更换主题？

安装

在博客根目录下执行如下命令：

$ git clone https://github.com/wuchong/jacman.git themes/jacman

启用

修改你的博客根目录下的_config.yml配置文件中的theme属性，将其设置为jacman。同时建议设置stylus属性中的compress值为true，会自动压缩 CSS 文件，hexo默认配置中不包含这一项，建议开启。如下

theme: jacman
stylus:
    compress: true

更新

cd themes/jacman
git pull origin master

请先备份您主题目录下的 _config.yml 文件后再升级。

参考链接

http://wuchong.me/blog/2014/11/20/how-to-use-jacman/
这里面有jacman主题配置的详细教程，大家可以参考里面的步骤操作。

发表文章

忙完了博客的搭建，其实最重要的还是内容，大家要把博客做好，关键是内容一定要丰富，而且质量要高，Hexo创建文章也特别简单。

在博客根目录下，执行命令：

$ hexo new "new article"

之后在source/_posts目录下面，多了一个new-article.md的文件

注意：新建、删除或修改文章后，不需要重启hexo server，刷新一下即可预览

相关问题

  1. 找不到git部署

    ERROR Deployer not found: git

    解决方法：

    npm install hexo-deployer-git --save

总结

搭建属于自己的博客确实是一件很爽的事情，hexo使用起来方便，搭配上github的远程服务，简直是绝配。不过这套框架需要我进一步验证，因为只用了几天，还没有发现什么问题，后续如果发现问题，我会不断的更新。
